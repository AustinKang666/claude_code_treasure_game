## Architecture

This is a single-page React + TypeScript app built with Vite. All game logic lives in one file: `src/App.tsx`.

**Game mechanics:** 3 treasure chests, one randomly assigned treasure. Clicking a chest reveals it — treasure gives +$100, skeleton gives -$50. Game ends when the treasure chest is found or all chests are opened.

**State shape in `App.tsx`:**
- `boxes: Box[]` — array of 3 boxes with `id`, `isOpen`, `hasTreasure`
- `score: number` — running score
- `gameEnded: boolean` — controls end-game overlay
- `mousePos: {x, y}` / `showKeyCursor: boolean` — drives a custom key-icon cursor that replaces the default pointer when hovering over closed chests

**Key dependencies:**
- `motion/react` (Framer Motion) — chest flip/scale animations
- `src/components/ui/` — shadcn/ui component library (Radix UI primitives + Tailwind)
- `src/components/figma/ImageWithFallback.tsx` — image component with SVG fallback on load error

**TypeScript:** transpiled by SWC via `@vitejs/plugin-react-swc` — there is no `typescript` package in devDependencies and no standalone `tsc` type-check step.

**Path alias:** `@` resolves to `./src` (configured in `vite.config.ts`). `vite.config.ts` also contains versioned package aliases (e.g. `vaul@1.1.2 → vaul`) generated by the Figma-to-code toolchain — do not remove them.

**Assets:**
- `src/assets/` — chest images (`treasure_closed.png`, `treasure_opened.png`, `treasure_opened_skeleton.png`, `key.png`)
- `src/audios/` — sound effects (`chest_open.mp3`, `chest_open_with_evil_laugh.mp3`)

**Build output** goes to `build/` (not the Vite default `dist/`).
